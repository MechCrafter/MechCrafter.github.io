obsidian 笔记备份历程

2024-03-25 · 1 min · 444 words · 随笔

原先是用 remotely-save 来在电脑和手机同步 obsidian 的笔记，那时候我刚接触编程，连 GitHub 都看不懂，不会用 GitHub，也不知道 git，只在网上看到有人说 git 可以版本控制，随时可以回滚笔记，和其它同步方法（百度网盘同步盘、syncthing 等）一起用可以防止丢笔记，但那时的我没有往那方面想，因为腾讯云的对象存储在官方文档中说对象存储支持版本记录，知道后来有一次无意间打开了腾讯云的 COSBrowser 的安装目录，发现所谓的版本记录是一份一份保存在本地，我才反应过来腾讯云那个版本记录是假的，我换台电脑就失效了

然后就去接触 git，但是 git 不支持云端加密，我有“网络洁癖”，不喜欢把笔记明文存到云端，于是乎又接触到了 git-crypt 于是又去学 git-crypt，搞来搞去稳定下来形成了一套 obsidian+git+git-crypt+gitee 的记笔记方法（用 gitee 是因为 gitee 国内网速稳定，GitHub 时而连的上时而连不上，我对笔记的要求是稳定）于是便舍弃了 remotely save，但今天 remotely save 更新了算法，最大限度的防丢笔记，原本心动了，想着 git+remotely save 可以双份备份笔记，但是 remotely save 毕竟不是官方同步，还是担心会搞乱笔记，于是还是用 git 来备份笔记，用 Duplicati 来定时同步，于是一套基于 obsidian 的笔记备份方法就形成了，git+gitee 来实时备份，Duplicati 来定时备份，每隔一段时间全库打包存到 OneDrive 和其它网盘里
